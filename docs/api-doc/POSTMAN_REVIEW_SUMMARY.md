## 🎓 Key Features Explained

### 1. Automatic Session Management

**What it does:**

- Stores session cookies automatically after login
- Sends cookies with every subsequent request
- No manual token management needed

**How to use:**

1. Login with any role
2. Cookies saved automatically
3. All future requests authenticated
4. Logout when done

**Benefits:**

- ✅ No manual work
- ✅ Consistent authentication
- ✅ Realistic browser behavior

---

### 2. Auto-Populated Variables

**What it does:**

- Saves IDs automatically after creation
- Uses saved IDs in related requests
- Updates dynamically

**Example Flow:**

```
1. Create Vehicle → ID saved to {{vehicle_id}}
2. Get Vehicle Details → Uses {{vehicle_id}}
3. Update Vehicle → Uses {{vehicle_id}}
4. Delete Vehicle → Uses {{vehicle_id}}
```

**Benefits:**

- ✅ No copy-paste needed
- ✅ Workflow automation
- ✅ Reduced errors

---

### 3. Built-in Tests

**What it does:**

- Validates status codes
- Checks response structure
- Verifies user roles
- Measures performance

**Example:**

```javascript
pm.test("Status code is 200", function () {
  pm.response.to.have.status(200);
});

pm.test("User role is admin", function () {
  var jsonData = pm.response.json();
  pm.expect(jsonData.user.role).to.eql("admin");
});
```

**Benefits:**

- ✅ Automatic validation
- ✅ Catch errors early
- ✅ Quality assurance

---

### 4. Collection Runner

**What it does:**

- Runs multiple requests sequentially
- Shows pass/fail statistics
- Generates test reports

**Use Cases:**

- Regression testing
- Smoke testing
- Load testing
- Automated testing

**Benefits:**

- ✅ Save time
- ✅ Consistent testing
- ✅ Detailed reports
